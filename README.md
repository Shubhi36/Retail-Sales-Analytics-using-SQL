Retail Sales Analytics using SQL
🧠 Project Summary

This project analyzes retail sales data to uncover valuable business insights such as yearly growth, customer losses, return rates,
shipping delays, discount impact, and salesperson performance.
The dataset was cleaned and transformed using SQL functions before performing analytical queries. The project demonstrates practical use of:

* JOINS
* CTEs (Common Table Expressions)
* Aggregate Functions
* Date Functions
* CASE Statements
* GROUP BY & ORDER BY
* Data Cleaning Techniques
* KPI Calculations

The analysis helps businesses improve decision-making by identifying:

* High-performing regions
* Loss-making customers
* Delivery-related return issues
* Seasonal sales trends
* Profitability by category
* Salesperson efficiency

📊 Key Business Problems Solved

1️⃣ Year-over-Year Sales & Profit Growth
Analyzed yearly sales and profit trends to track business growth.

2️⃣ Weekend Sales Effect
Studied how sales and returns vary across weekdays.

3️⃣ Best Performing Quarter
Identified the quarter with maximum revenue generation.

4️⃣ Discount vs Profit Analysis
Found categories where high discounts reduce profitability.

5️⃣ Return Rate Analysis
Calculated return percentages region-wise.

6️⃣ Loss-Making Customers
Identified customers generating negative profits.

7️⃣ Shipping Delay Analysis
Measured average shipping delays for different shipping modes.

8️⃣ Late Delivery Impact
Analyzed whether delayed deliveries increase product returns.

9️⃣ Salesperson Performance Tracking
Compared salespersons based on sales and profit margins.
